# 📋 프론트엔드 개발·디자인 세부 계획서 커피라이크 AI 시크릿 타로

* **기술 스택 (권장):** HTML5, CSS3, Vanilla JS (또는 React/Vue 등 가벼운 프레임워크), AJAX/Fetch API (백엔드 통신용)

## 3\. 페이지별 세부 스토리보드 및 개발 요건 (프론트엔드 개발자 용)

### 📄 Page 1: 랜딩 및 인증 화면 (/index.html)

테이블 QR코드를 찍고 들어오는 첫 화면입니다.

* **UI/UX 디자인:**  
* 상단: '커피라이크' 로고 및 "당신의 오늘을 읽어드립니다" 환영 문구.  
* 중앙: \[전화번호 뒤 4자리\] 입력 폼 (InputBox).  
* 하단: \[오늘의 무료 운세 보기\] 버튼 (Primary Color).  
* **프론트엔드 로직 (기능 구현):**  
* 입력 폼은 **숫자만 4자리** 입력되도록 유효성 검사(Validation) 처리.  
* QR코드 URL의 쿼리 파라미터(예: ?table=1)를 파싱하여 숨김 값(Hidden Field)으로 유지.  
* 버튼 클릭 시 백엔드 API (/api/auth)로 데이터 전송.

### 📄 Page 2: 무료 운세 결과 & 심층 타로 업셀링 (핵심 화면, /result-basic.html)

기본 운세를 보여주고, 심층 타로 결제를 유도하는 화면입니다.

* **UI/UX 디자인:**  
* 상단: 짧은 무료 운세 텍스트 결과 노출.  
* 중앙 (업셀링 영역): 타로 카드 이미지가 **흐리게(CSS Blur 효과)** 처리되어 호기심 자극.  
* **포인트 게이지 바 (게이미피케이션 요소):**  
* "현재 고객님의 포인트는 API수신값P\! 타로를 보려면 남은값P가 더 필요해요."  
* 게이지 바 애니메이션 적용 (0에서 현재 포인트 퍼센트까지 차오르는 효과).  
* 하단 (결제 버튼 \- 크기와 색상으로 대비 줌):  
* **버튼 A (강조/테크 블루):** ✨ 내 포인트로 싸게 보기 (2,000P 차감) \-\> *단, 3,000P 이상일 때만 사용 가능함 명시.*  
* **버튼 B (일반/회색):** 💳 일반 결제 (3,000원)  
* **프론트엔드 로직 (기능 구현):**  
* 백엔드에서 받아온 고객의 현재 보유 포인트(current\_point)를 변수에 저장.  
* **버튼 A 클릭 이벤트 분기 처리 (핵심 로직):**  
* if (current\_point \>= 3000\) \-\> 대기 번호 발급 API 호출 후 Page 3으로 이동.  
* else if (current\_point \< 3000\) \-\> 팝업 경고창(Alert/Modal) 띄움. *"포인트가 부족해요\! 카운터에서 디저트를 추가 결제하시면 바로 타로를 보실 수 있습니다."* (추가 결제 유도).  
* **버튼 B 클릭 시:** 대기 번호 발급 API 호출 후 Page 3으로 이동.

### 📄 Page 3: 결제 대기 / 웨이팅 화면 (/waiting.html)

고객이 카운터로 가서 결제/차감하는 동안 띄워두는 대기 화면입니다.

* **UI/UX 디자인:**  
* 중앙 화면에 크게 **"고객님의 요청 번호: A-12"** 노출.  
* 하단 안내 텍스트: "카운터에 번호를 알려주시면 사장님이 타로 카드를 열어드립니다."  
* 로딩 애니메이션 (타로 카드가 셔플되는 GIF 등) 배치.  
* **프론트엔드 로직 (기능 구현):**  
* **Polling(폴링) 구현:** setInterval을 사용하여 3초마다 백엔드 API (/api/check-status)를 호출해 사장님이 포스기 처리 후 관리자 페이지에서 승인 버튼을 눌렀는지 체크.  
* 응답 상태값(status \== 'approved')을 받으면 자동으로 Page 4로 리다이렉트(Redirect) 처리.

### 📄 Page 4: 심층 타로 결과 및 공유 화면 (/tarot-result.html)

사장님 승인 후 열리는 최종 결과 화면입니다.

* **UI/UX 디자인:**  
* 화면 진입 시 타로 카드가 화려하게 뒤집히는 **CSS 3D Flip 애니메이션** 적용.  
* AI가 작성한 장문의 심층 타로 결과 텍스트 (가독성을 위해 자간/행간 여유 있게 설정, 폰트: 프리텐다드 2).  
* 하단 강조 버튼: \[인스타그램에 공유하고 500P 적립받기\] 4, 5\.  
* **프론트엔드 로직 (기능 구현):**  
* 백엔드에서 생성된 타로 결과(이미지 URL 및 텍스트)를 렌더링.  
* 공유 버튼 클릭 시, 모바일 기기의 네이티브 공유 API(navigator.share())를 호출하거나 인스타그램 앱을 띄울 수 있는 딥링크(Deep Link) 연결.

## 4\. 백엔드 통신 API 명세 (참고용 List)

개발자가 프론트엔드 로직을 짤 때 필요한 API 목록입니다. (대표님께서 ASP/SQL로 구성하실 백엔드 응답 규격입니다.)

1. POST /api/auth : 폰번호 4자리 전송 ➡️ 응답: 무료운세 텍스트, 현재 포인트 잔액  
2. POST /api/request-tarot : 사용자가 선택한 결제방식(포인트/일반) 전송 ➡️ 응답: 대기번호(예: A-12)  
3. GET /api/check-status?req\_id=A-12 : 사장님 승인 여부 지속 체크 ➡️ 응답: 상태값 (대기중/승인완료)  
4. GET /api/get-result?req\_id=A-12 : 승인 완료 시 심층 타로 결과 데이터 수신
