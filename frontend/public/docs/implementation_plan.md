# 📋 최종 구현 계획서: 커피라이크 타로 개발 (V3)

대표님의 최신 지시사항(7차 음성 명령)을 완벽히 반영한 최종 계획서입니다. 비용 0원의 오픈소스 데이터 전략과 Supabase 백엔드, 그리고 '커피라이크'만의 영수증 인증 로직을 핵심으로 구축합니다.

## 👥 사용자 검토 필요 (User Review Required)

> [!IMPORTANT]
> **기술 스택 및 인증 방식 확정:** 
> 1. **프론트엔드:** React + Vite + Tailwind CSS
> 2. **백엔드:** **Supabase** (DB 및 인증 관리)
> 3. **데이터:** AI 연동 없이 **오픈소스 타로 데이터셋** 활용 (비용 절감)
> 4. **인증:** 전화번호 대신 **영수증 번호(YYYYMMDD + 01 + 시리얼 4자리)**의 마지막 4자리 사용

## 🛠️ 주요 개발 항목

### 1단계: 프로젝트 기초 공사
- [ ] `CoffeeLike_Tarot/frontend`: Vite + React 환경 세팅
- [ ] **Supabase 프로젝트 설정:** 테이블 설계 (`TB_Customer`, `TB_Tarot_Request`) 및 초기화
- [ ] **에셋 생성:** `generate_image`를 통해 '커피라이크 타로' 전용 카드 이미지(22종) 생성

### 2단계: 프론트엔드 핵심 기능 (커피라이크 타로 전용)
- [ ] **인증 로직:** 영수증 번호 뒤 4자리 입력 폼 및 유효성 검사 (입력값 -> DB 조회)
- [ ] **데이터 연동:** 오픈소스 타로 데이터셋(JSON) 파싱 및 랜덤 카드 뽑기 로직
- [ ] **UI/UX:** Stitch 디자인 시스템 적용 (Coffee Brown & Tech Blue), 1px 선 지양, 톤 차이에 의한 구분

### 3단계: 페이지별 상세 구현
- [ ] **Page 1 (랜딩):** 영수증 번호 입력 및 당일 날짜 기반 로직
- [ ] **Page 2 (결과/업셀링):** 오픈소스 기반 기본 해석 출력, 포인트 허들 경고 팝업
- [ ] **Page 3 (웨이팅):** Supabase 실시간(Realtime) 구독 기능을 활용한 사장님 승인 대기
- [ ] **Page 4 (최종 결과):** 3D 카드 플립 연출 및 상세 오픈소스 데이터 출력
