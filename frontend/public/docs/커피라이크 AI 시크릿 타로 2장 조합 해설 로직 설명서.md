# 🔮 커피라이크 AI 시크릿 타로: 2장 조합 해설 로직 설명서

본 문서는 커피라이크 타로 서비스의 핵심 가치인 '2장 카드 조합을 통한 입체적 운세 분석' 시스템의 비즈니스 로직과 기술적 구현 방안을 정의합니다.

---

## 1. 비즈니스 워크플로우 (User Experience)

### 1단계: 운명의 시작 (무료 드로우)
- **액션**: 고객이 휴대폰 번호 입력 후 접속하여 첫 번째 카드(`Card 1`)를 뽑습니다.
- **의미**: 현재의 에너지, 혹은 과거로부터 이어진 현재의 상황(Past/Present)을 상징합니다.
- **비용**: **무료 (Free)**
- **결과**: 해당 카드의 기본 키워드와 가벼운 오늘의 신탁(Oracle) 정보를 즉시 노출합니다.
## 1. 개요

'커피라이크 타로' 서비스의 핵심 차별화 전략으로, 사용자가 첫 번째 무료 카드를 뽑은 후 **더 깊은 통찰(Deep Insight)**을 원할 경우 유료 포인트(3,000P)를 사용하여 두 번째 카드를 추가로 뽑고, 두 카드의 에너지를 결합한 **고도화된 AI 심층 해설**을 제공하는 시스템입니다.

---

## 2. 사용자 시나리오 (User Journey)

1. **카드 드로우 (무료)**: 사용자는 평소처럼 첫 번째 타로 카드를 뽑고 기본 해설을 확인합니다.
2. **심층 타로 신청 (하이패스)**: 결과 화면 하단의 '심층 타로 신청' 버튼을 클릭합니다.
3. **자동 카드 매칭 및 요청**: 시스템이 즉시 두 번째 운명을 결정(자동 추첨)하고, Supabase RPC `process_deep_tarot_request`를 통해 포인트 확인 및 요청 생성을 한 번에 처리합니다.
4. **바리스타(관리자) 승인 대기**: 사용자는 별도의 조작 없이 즉시 '바리스타 승인 대기' 화면(대기 번호 노출)으로 이동합니다.
5. **바리스타(관리자) 승인**: 관리자 페이지에서 두 장의 카드 조합을 확인하고 승인합니다.
6. **최종 결과 확인**: 바리스타 승인 시 실시간으로 결과가 노출됩니다. 두 카드가 나란히 배치된 모습과 함께, 두 기운이 섞인 특별한 심층 해설을 확인합니다.

---

## 3. 기술적 구현 세부 사항 (Technical Details)

### 데이터베이스 (Supabase)

- **테이블 변경**: `tb_tarot_request` 테이블에 `tarot_card2_name` 컬럼을 신규 추가하여 두 번째 카드 정보를 저장합니다.
- **Atomic Transaction**: 신규 RPC 함수(`process_deep_tarot_request`)를 통해 '포인트 확인 -> 포인트 차감 -> 요청 생성' 과정을 하나의 트랜잭션으로 처리하여 데이터 무결성을 보장합니다.

### 프론트엔드 (React + Tailwind CSS)

- **상태 관리**: `selectedCard2` (두 번째 카드), `isCasting2` (드로우 상태), `drawing2` (전용 UI 화면) 상태를 추가하여 복합적인 사용자 흐름을 제어합니다.
- **UI 컴포넌트**: `TarotCard` 컴포넌트에 `size="small"` 속성을 추가하여 결과 화면에서 두 장을 조화롭게 배치(Grid)합니다.
- **실시간 폴링**: 첫 번째 카드 요청 때와 동일하게 Supabase 실시간 구독 또는 폴링 시스템을 통해 관리자의 승인 여부를 실시간으로 감지합니다.

---

## 4. AI 심층 해설 로직 (AI Logic)

현재는 사전에 정의된 78장의 `deep_interpretation` 데이터를 활용하여 두 카드의 해설을 조합합니다.

- **Fortune**: 카드 1의 현재 기운 + 카드 2의 미래 기운 결합
- **Insight**: 두 카드의 영적 통찰 메시지 병합
- **Caution**: 두 카드의 주의사항을 모두 포함하여 예방 효과 극대화
- **Coffee Pairing**: 두 카드의 에너지를 중화하거나 강화할 수 있는 마스터 추천 커피 제시

---

## 5. 관리자 대시보드 (Barista Dashboard)

바리스타(관리자)는 신청 리스트에서 즉시 **'Card1 + Card2'** 형태의 조합을 시각적으로 확인하여, 사용자의 고민을 더 입체적으로 파악하고 정성스러운 승인 처리를 진행할 수 있습니다.
